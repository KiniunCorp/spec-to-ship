# Roadmap

This roadmap reflects the current direction of Spec-To-Ship.

It is intentionally lightweight and will evolve over time.

---

## Now

Focus on stabilizing and strengthening the core:

- CLI reliability and correctness
- stage orchestration and workflow clarity
- governance files and project structure
- documentation completeness and clarity
- onboarding experience (`s2s init`, `s2s doctor`)

---

## Next

Expand usability and real-world adoption:

- improved examples and real-world use cases
- better documentation for different user profiles (solo dev, team, platform)
- improved developer experience and ergonomics
- deeper integration patterns with common AI workflows
- visibility and discoverability improvements

---

## Later

Longer-term exploration areas:

- advanced workflow extensions
- richer project state introspection and visualization
- integration patterns for team environments
- optional ecosystem tooling around Spec-To-Ship

---

## Principles

The roadmap is guided by:

- keeping the tool genuinely useful as OSS
- avoiding unnecessary complexity
- prioritizing clarity and execution quality
- maintaining a clean separation from future commercial products

---

## Important notes

- Features are not prioritized based on sponsorship alone
- No functionality is intentionally restricted to paid users
- Commercial work (if any) will be built separately, not by degrading the OSS version

---

## Feedback

Feedback is welcome through GitHub Issues.

Not all requests will be implemented, but useful patterns and repeated needs will influence prioritization.
